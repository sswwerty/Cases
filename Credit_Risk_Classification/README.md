# Credit Risk Scoring (Binary Classification)

## Задача и зачем она решалась

Цель проекта — построить модель кредитного скоринга, которая прогнозирует вероятность дефолта клиента.  
Такая задача является классической для финансовых и банковских систем и напрямую связана с бизнес-рисками:
ошибка модели может приводить либо к финансовым потерям (выдача кредита ненадёжному клиенту), либо к упущенной прибыли (отказ надёжному заёмщику).

Проект выполнен с фокусом на:
- работу с табличными данными,
- корректную валидацию моделей,
- борьбу с переобучением,
- интерпретируемость решений.

---

## Что было сделано

### 1. Подготовка данных
- выполнен первичный анализ распределений признаков;
- обработаны пропущенные значения;
- данные разделены на обучающую и валидационную выборки без утечки информации.

### 2. Построение baseline-моделей
Для понимания сложности задачи и поведения данных были обучены:
- Decision Tree  
- Random Forest  

Это позволило:
- оценить переобучение,
- понять предел качества без сложных ансамблей.

### 3. Финальная модель — XGBoost
В качестве основной модели был выбран **XGBoost**, так как он хорошо работает с табличными данными и позволяет контролировать bias–variance trade-off.

Ключевые моменты обучения:
- учтён **дисбаланс классов** с помощью `scale_pos_weight`;
- использовано **early stopping** для предотвращения переобучения;
- подобраны основные гиперпараметры (`max_depth`, `learning_rate`, `subsample`, `colsample_bytree`) с объяснением их влияния;
- модель обучалась строго на train-части, валидация использовалась только для контроля качества.

### 4. Оценка качества
- модель оценивалась с использованием метрик, устойчивых к дисбалансу классов (ROC-AUC, F1);
- дополнительно проанализирована важность признаков, чтобы убедиться в отсутствии явной утечки данных и логичности решений модели.

---

## Результат и выводы

- XGBoost показал стабильное и  лучшее качество по сравнению с baseline-моделями;
- использование early stopping позволило сохранить обобщающую способность модели;
- наиболее значимые признаки соответствуют ожиданиям для задачи кредитного скоринга, что повышает доверие к модели;
- проект демонстрирует полный ML-пайплайн: от baseline до финальной модели с контролем переобучения.
